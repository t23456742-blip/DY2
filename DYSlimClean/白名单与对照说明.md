### 为什么会这样

1. **旧版 App 白名单错了**  
   之前内置的是 `H9-20-土` **全部 20 个 zip 的并集**（约 3140 条），比你指定的 `20260106-13-20-04_1.zip` 宽很多。  
   所以很多「相对这个单包多余、但落在并集里」的文件被当成可保留，**没有删**。

2. **精确路径才算保留**  
   例如表情包目录带 hash：
   - 参考：`Library/AWEIMRoot/Sticker/Little/3d0cbda7.../`
   - 手机：`Library/AWEIMRoot/Sticker/Little/7a3d4dd8.../`  
   文件名相同也算**不同路径**，按规则应删除。旧并集白名单更容易把这类路径留住。

3. **多余文件主要堆在**（删除后备份统计）

   | 目录 | 约多余数 |
   |------|----------|
   | `Library/Pitaya` | 1864 |
   | `Library/Application Support` | 1236 |
   | `Library/AWEIMRoot` | 697 |
   | `Documents/...`（IM/动画等） | 434 |
   | `Library/Caches` | 196 |
   | `Library/tma` 等 | 其余 |

## 正确用法（重新编译后）

1. 已把 `keep_paths.txt` **改成只含** `20260106-13-20-04_1.zip` 的路径。  
2. 把更新后的工程推到 GitHub，再跑 **Build TIPA**，巨魔重装新 tipa。  
3. 建议：先完全退出抖音 → 打开「抖音精简清理」→ **扫描** → 看「多余」数量（应接近四千级）→ **一键删除**。  
4. 再用 Filza 导出容器，对比参考包文件数应接近 **1667**（抖音运行后会再长出少量缓存，属正常）。
